Pulse supports 7 application modes, each with a dedicated generator, runner,
config manager, and response converter. This document covers the app mode
taxonomy, the execution pipeline from API request to workflow execution, and
the task pipeline for streaming responses.

## App Modes

Defined in `api/models/model.py`:

```python
class AppMode(StrEnum):
    COMPLETION = "completion"        # Single-turn text completion
    WORKFLOW = "workflow"            # Standalone workflow execution
    CHAT = "chat"                    # Simple chatbot
    ADVANCED_CHAT = "advanced-chat"  # Chatbot backed by workflow
    AGENT_CHAT = "agent-chat"       # Agent-based chatbot
    CHANNEL = "channel"             # Channel integrations
    RAG_PIPELINE = "rag-pipeline"   # Document processing pipeline
```

### Mode Characteristics

| Mode | Workflow Engine | Memory | Streaming | Multi-turn |
| ---- | -------------- | ------ | --------- | ---------- |
| COMPLETION | No | No | Yes | No |
| CHAT | No | Yes | Yes | Yes |
| WORKFLOW | Yes | No | Yes | No |
| ADVANCED_CHAT | Yes | Yes | Yes | Yes |
| AGENT_CHAT | No | Yes | Yes | Yes |
| CHANNEL | Yes | Yes | Yes | Yes |
| RAG_PIPELINE | Yes | No | Yes | No |

## Base Classes

### BaseAppGenerator

`api/core/app/apps/base_app_generator.py` -- Handles input preparation:

```python
class BaseAppGenerator:
    def _prepare_user_inputs(
        self,
        *,
        user_inputs: Mapping[str, Any] | None,
        variables: Sequence[VariableEntity],
        tenant_id: str,
        strict_type_validation: bool = False,
    ) -> Mapping[str, Any]:
        # 1. Filter inputs by form configuration
        # 2. Validate required fields, defaults, option values
        # 3. Convert file inputs to File objects
        # 4. Convert file list inputs
        # 5. Return sanitized inputs
```

### AppRunner (Easy UI modes)

`api/core/app/apps/base_app_runner.py` -- For completion/chat/agent_chat
(non-workflow modes):

```python
class AppRunner:
    def recalc_llm_max_tokens(self, model_config, prompt_messages):
        """Recalculate max_tokens if prompt + max exceeds context window."""

    def organize_prompt_messages(self, app_record, model_config,
        prompt_template_entity, inputs, ...):
        """Build prompt messages from template + inputs + memory."""
```

## Execution Pipeline

### Workflow Mode

```mermaid
graph TD
    Controller["Controller\n(POST /workflows/run)"] --> Generator["WorkflowAppGenerator.generate()"]
    Generator --> Load["1. Load Workflow from DB"]
    Generator --> Validate["2. Validate app config\n(WorkflowAppConfigManager)"]
    Generator --> Prepare["3. Prepare user inputs\n(_prepare_user_inputs)"]
    Generator --> CreateEntity["4. Create WorkflowAppGenerateEntity"]
    Generator --> Spawn["5. Spawn background thread"]
    Spawn --> Runner["WorkflowAppRunner.run()"]
    Runner --> BuildGraph["Build Graph from workflow config"]
    Runner --> CreatePool["Create VariablePool with inputs + system vars"]
    Runner --> CreateState["Create GraphRuntimeState"]
    Runner --> AttachLayers["Attach layers (ExecutionLimits, Observability, ...)"]
    Runner --> CreateEntry["Create WorkflowEntry"]
    Runner --> Execute["Execute GraphEngine.run()"]
    Runner --> ConvertEvents["Convert GraphEngineEvents -> QueueEvents"]
    Runner --> PushEvents["Push events to AppQueueManager"]
    Generator --> ReturnPipeline["6. Return WorkflowAppGenerateTaskPipeline"]
    ReturnPipeline --> StreamSSE["Stream QueueEvents -> SSE Response"]
```

### Advanced Chat Mode

```mermaid
graph TD
    ChatController["Controller\n(POST /chat-messages)"] --> ChatGenerator["AdvancedChatAppGenerator.generate()"]
    ChatGenerator --> LoadWF["1. Load Workflow from DB"]
    ChatGenerator --> LoadConv["2. Load or create Conversation"]
    ChatGenerator --> CreateMsg["3. Create Message record"]
    ChatGenerator --> LoadHistory["4. Load conversation history (memory)"]
    ChatGenerator --> PrepInputs["5. Prepare user inputs"]
    ChatGenerator --> CreateChatEntity["6. Create AdvancedChatAppGenerateEntity"]
    ChatGenerator --> SpawnChat["7. Spawn background thread"]
    SpawnChat --> ChatRunner["AdvancedChatAppRunner.run()"]
    ChatRunner --> SameAsWF["Same as WorkflowAppRunner but with:\n- Conversation variables loaded\n- dialogue_count set\n- Message updated with outputs"]
    ChatGenerator --> ReturnChatPipeline["8. Return AdvancedChatAppGenerateTaskPipeline"]
    ReturnChatPipeline --> ChatStreamSSE["Stream QueueEvents -> SSE Response"]
```

### RAG Pipeline Mode

```mermaid
graph TD
    PipelineCtrl["Controller or Celery Task"] --> PipelineGen["PipelineGenerator.generate()"]
    PipelineGen --> LoadPipeline["1. Load Pipeline + Workflow"]
    PipelineGen --> PrepDocMeta["2. Prepare document metadata\nas system variables\n(document_id, dataset_id, datasource_type, batch, ...)"]
    PipelineGen --> CreateRAGEntity["3. Create RagPipelineGenerateEntity"]
    PipelineGen --> SpawnPipeline["4. Spawn background thread"]
    SpawnPipeline --> PipelineRunner["PipelineRunner.run()"]
    PipelineRunner --> SimilarWF["Similar to WorkflowAppRunner"]
    PipelineRunner --> DatasourceRoot["DATASOURCE node as root\ninstead of START"]
    PipelineRunner --> RAGVars["RAG-specific system variables"]
    PipelineGen --> ReturnResult["Return task pipeline or result"]
```

## Task Pipeline for Streaming

The task pipeline converts internal queue events into SSE response events
for the frontend.

### Queue Event Flow

```mermaid
graph TD
    GEEvents["GraphEngine Events"] --> HandleEvent["WorkflowBasedAppRunner._handle_event()\nConverts GraphEngineEvent -> QueueEvent"]
    HandleEvent --> QueueMgr["AppQueueManager.publish(event)\nThread-safe queue.Queue"]
    QueueMgr --> TaskPipeline["GenerateTaskPipeline.process()\nConsumes QueueEvents\nConverts to response format"]
    TaskPipeline --> SSEStream["SSE Response Stream\nworkflow_started -> node_started ->\ntext_chunk -> workflow_finished"]
```

### Chat Request and Workflow Execution Flow

```mermaid
sequenceDiagram
    participant Client
    participant Controller
    participant Generator as AppGenerator
    participant Runner as AppRunner
    participant Engine as GraphEngine
    participant Queue as AppQueueManager
    participant Pipeline as TaskPipeline

    Client->>Controller: POST /chat-messages
    Controller->>Generator: generate(inputs)
    Generator->>Generator: Load workflow, validate config
    Generator->>Generator: Create conversation + message
    Generator-->>Controller: TaskPipeline (streaming)

    Note over Generator,Runner: Background thread
    Generator->>Runner: run()
    Runner->>Engine: GraphEngine.run()
    loop For each node
        Engine->>Engine: Execute node
        Engine-->>Runner: GraphEngineEvent
        Runner->>Queue: publish(QueueEvent)
    end
    Queue-->>Pipeline: consume events
    Pipeline-->>Client: SSE stream events
```

## Response Converters

Each app mode has a response converter that formats the final response:

```python
class WorkflowAppGenerateResponseConverter:
    @classmethod
    def convert_blocking(cls, result) -> WorkflowAppBlockingResponse: ...

    @classmethod
    def convert_stream(cls, generator) -> Generator[WorkflowAppStreamResponse]: ...
```

### Blocking vs Streaming

| Mode | Blocking Response | Streaming Response |
|------|-------------------|-------------------|
| Workflow | `WorkflowAppBlockingResponse` | `WorkflowAppStreamResponse` |
| Advanced Chat | `ChatbotAppBlockingResponse` | `ChatbotAppStreamResponse` |
| Completion | `CompletionAppBlockingResponse` | `CompletionAppStreamResponse` |
