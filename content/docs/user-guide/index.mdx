Welcome to the Pulse User Guide. This guide is for anyone who wants to use
Pulse to build AI-powered applications, whether you are a product manager,
a founder, a business analyst, or simply curious about what Pulse can do.

No programming experience is required. Every concept is explained in plain
language.

## How to Use This Guide

- **New to Pulse?** Start with Chapter 01 and work through in order.
- **Want to build something specific?** Jump to [Recipes](/docs/user-guide/recipes)
  and find a guide that matches your goal.
- **Something not working?** Check [Troubleshooting](/docs/user-guide/troubleshooting-faq).
- **Want to go deeper?** The [Architecture docs](/docs/architecture) are for
  engineers who want to understand how Pulse works internally.

---

## Conventions in This Guide

- **Bold terms** are defined the first time they appear.
- Step-by-step instructions use numbered lists (1, 2, 3...).
- Helpful tips appear as blockquotes.
- No code or technical jargon -- everything is explained in everyday language.

---

## Quick Glossary

| Term | Simple Explanation |
|------|--------------------|
| App | A complete AI application that you build and share with others |
| Workflow | A visual diagram that tells the AI what steps to follow |
| Node | A single step in a workflow (like a box in a flowchart) |
| Model | The AI brain that generates text, images, or other content |
| Knowledge Base | A collection of your documents that the AI can search |
| Plugin | An add-on that gives Pulse extra abilities |
| Workspace | Your team's shared area where all apps and data live |
