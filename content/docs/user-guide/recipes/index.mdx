Recipes are step-by-step guides for building specific types of
applications in Pulse. Each recipe follows the same structure:

1. **Goal** -- what you will build
2. **Prerequisites** -- what you need before starting
3. **Steps** -- detailed instructions
4. **Testing** -- how to verify it works
5. **Variations** -- ideas for customization

---

## Which Recipe Should I Start With?

- **"I want a chatbot that knows my company's information"**
  --> Start with [RAG Chatbot](/docs/user-guide/recipes/rag-chatbot)

- **"I want a customer support system"**
  --> Start with [Customer Support Bot](/docs/user-guide/recipes/customer-support-bot)

- **"I want to generate content automatically"**
  --> Start with [Content Generation Pipeline](/docs/user-guide/recipes/content-generation-pipeline)

- **"I want to connect Pulse to other tools"**
  --> Start with [Webhook-Triggered Automation](/docs/user-guide/recipes/webhook-triggered-automation)

- **"I want a workflow that runs on a schedule"**
  --> Start with [Scheduled Workflow](/docs/user-guide/recipes/scheduled-workflow)

## Before Starting Any Recipe

Make sure you have:

1. A Pulse account with Editor or higher permissions.
2. At least one model provider configured (see
   [Model Configuration](/docs/user-guide/model-configuration)).
3. Completed [Building Your First Chatbot](/docs/user-guide/building-your-first-chatbot)
   so you understand the basics.
