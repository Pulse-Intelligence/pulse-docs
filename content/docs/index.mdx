Welcome to the Pulse documentation hub. Pulse is a fork of [Dify](https://github.com/langgenius/dify) --
an open-source platform for building LLM-powered applications with an intuitive
visual interface, agentic workflows, RAG pipelines, and plugin-based model
management.

## How to Use These Docs

The documentation is organized into four tiers, each aimed at a different
audience and level of depth. Start with whichever tier matches your current
need and follow the cross-references to go deeper.

## Conventions Used in These Docs

Every document follows these conventions:

1. **Audience tag** -- Each file starts with an audience and prerequisites line.
2. **Pulse branding** -- We use "Pulse" throughout. "Dify" appears only when
   explaining our relationship to the upstream fork.
3. **No secrets** -- Placeholder values like `YOUR_API_KEY` are used instead of
   real credentials.
4. **Cross-references** -- Documents link to each other with relative paths.
5. **ASCII diagrams** -- Non-trivial flows are illustrated with ASCII art.
6. **Code examples** -- Drawn from the actual codebase, not hypothetical.
